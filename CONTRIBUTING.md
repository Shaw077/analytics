# Contributing Guidelines

Thanks for your interest in contributing.

## Workflow

1. Fork the repository and create a branch from `main`.
2. Use a clear branch name, like `feature/add-auth` or `fix/login-bug`.
3. Keep pull requests focused and small.
4. Add or update tests for behavior changes.
5. Update docs when needed.

## Pull Request Checklist

- [ ] Code builds and tests pass.
- [ ] New behavior is tested.
- [ ] README/docs are updated when needed.
- [ ] PR description explains the change and impact.

## Commit Messages

Use concise, descriptive commit messages. Prefer:

- `feat: add user profile endpoint`
- `fix: handle null session in auth flow`
- `docs: update setup instructions`

## Code Style

Follow existing style and naming patterns in this repository.
